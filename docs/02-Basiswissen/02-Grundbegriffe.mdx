# Fachbegriffe in der Kommunikation

In der Kommunikationstechnik gibt es eine Reihe von Fachbegriffen, die beschreiben, **wie Daten übertragen und verteilt werden**. Dieses Kapitel behandelt zentrale Konzepte, darunter Leitungs- und Paketvermittlung, Übertragungsrichtungen, Multiplexing und Adressierungsformen.

---

## Leitungsvermittlung und Paketvermittlung

<Columns>
  <Column className='text--left'>
    #### Leitungsvermittlung
    Bei der Leitungsvermittlung wird für die Dauer einer **Kommunikationssitzung** eine feste **Verbindung** zwischen Sender und Empfänger aufgebaut. 
    - Vorteil: konstante Bandbreite, geringe Latenz.
    - Nachteil: ineffizient bei sporadischem Datenverkehr.

    Merkhilfe: Wie ein **Telefonat**, bei dem eine Leitung reserviert wird. Wenn die Sprechenden einschlafen, bleiben die Ressourcen reserviert.
  </Column>
  <Column className='text--left'>
    #### Paketvermittlung
    Daten werden in **Pakete** zerlegt und unabhängig voneinander über ein Netzwerk gesendet.
    - Vorteil: effizient, robust bei Ausfällen.
    - Nachteil: Pakete können in falscher Reihenfolge ankommen, zusätzliche Verarbeitung nötig. Jede Einheit muss Anfang, Ende und Zieladresse haben.

    Merkhilfe: Wie **Briefe per Post**, die unabhängig voneinander verschickt werden.
  </Column>
</Columns>

## Übertragungsrichtungen: unidirektional vs. bidirektional

<Columns>
  <Column className='text--left'>
    #### Unidirektional
    Daten fließen nur in eine Richtung. Beispiele: Radio, TV.

    Merkhilfe: **Einbahnstraße**.
  </Column>
  <Column className='text--left'>
    #### Bidirektional
    Daten können in beide Richtungen fließen. Beispiele: Telefon, Chat.

    Merkhilfe: Wie ein **Zweispurige Straße, zwei Fahrtrichtungen**.
  </Column>
</Columns>

---

## Multiplexing: Simplex, Halbduplex, Vollduplex

<Columns>
  <Column className='text--left'>
    #### Simplex
    Daten fließen nur in **eine Richtung**, ohne Rückmeldung. 
    Beispiel: Fernsehsender.

    Merkhilfe: Ein **Radio** kann nur empfangen.
  </Column>
  <Column className='text--left'>
    #### Halbduplex
    Daten fließen **abwechselnd** in beide Richtungen. 
    Beispiel: Walkie-Talkie.

    Merkhilfe: Einer spricht, der andere hört zu, dann umgekehrt.
  </Column>
  <Column className='text--left'>
    #### Vollduplex
    Daten fließen **gleichzeitig** in beide Richtungen. 
    Beispiel: modernes Telefon oder VoIP.

    Merkhilfe: Wie ein **normales Gespräch**, beide können gleichzeitig sprechen.
  </Column>
</Columns>

## Adressierungsformen: unicast, broadcast, multicast, anycast

<Columns>
  <Column className='text--left'>
    #### Unicast
    Daten werden von einem Sender **an genau einen Empfänger** gesendet.
    Beispiel: Private E-Mail.
  </Column>
  <Column className='text--left'>
    #### Broadcast
    Daten werden von einem Sender **an alle Empfänger** im Netzwerk gesendet.
    Beispiel: ARP-Request im LAN.
  </Column>
  <Column className='text--left'>
    #### Multicast
    Daten werden an eine **bestimmte Gruppe von Empfängern** gesendet.
    Beispiel: Video-Konferenz.
  </Column>
  <Column className='text--left'>
    #### Anycast
    Daten werden an **den nächstgelegenen von mehreren möglichen Empfängern** gesendet.
    Beispiel: DNS-Abfrage zu einem nahegelegenen Server.
  </Column>
</Columns>

## Datenübertragung: Parallel vs. Seriell
<Columns>
<Column className='text--left'>
#### Parallel
Daten werden **gleichzeitig über mehrere Leitungen** übertragen.  
- Vorteil: hohe Übertragungsrate bei kurzen Distanzen.  
- Nachteil: Störungen und Signalverluste über lange Strecken.  
Merkhilfe: Wie mehrere **Autospuren gleichzeitig**, auf denen Autos parallel fahren.
</Column>

<Column className='text--left'>
#### Seriell
Daten werden **nacheinander über eine einzige Leitung** übertragen.  
- Vorteil: zuverlässig über längere Strecken, weniger Leitungen nötig.  
- Nachteil: geringere Geschwindigkeit im Vergleich zu Parallel bei kurzen Distanzen.  
Merkhilfe: Wie **Autos nacheinander auf einer Einbahnstraße**.
</Column>
</Columns>
